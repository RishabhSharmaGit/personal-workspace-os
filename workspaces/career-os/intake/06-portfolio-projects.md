# 06 — Portfolio & projects

What's worth showing externally? List projects with measurable impact, public artifacts, or unusual technical depth. For each, you'll later promote to a file under `projects/`.

## Three-tier portfolio model (industry standard for senior SWEs in 2026)

1. **GitHub profile README** — pinned 3-5 repos, brief bio, contact. Recruiters glance at contribution graph.
2. **Personal site** — case studies with metrics, architecture diagrams, talks/posts. Custom `.dev` or `.com` domain on Vercel/Netlify/GH Pages.
3. **3-5 deep projects with live demos** — these are what you'd send to a hiring manager.

## Project promotion checklist

Before promoting a project to `projects/{slug}.md`, it should have:

- [ ] A 1-sentence summary you could put on a resume
- [ ] At least 1 quantified metric (users / RPS / latency / revenue / etc.)
- [ ] A description of one specific trade-off you navigated
- [ ] An evidence link (repo / demo / case study / screenshot)
- [ ] Tech stack tags

Without those, it's not really a portfolio piece — it's just a bullet.
